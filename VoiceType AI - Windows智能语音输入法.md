# VoiceType AI - Windows 智能语音输入法

## 一句话描述

一个 Windows 下的快捷键触发的智能语音输入法，支持实时转写、多风格润色、本地/云端模型切换，最终通过剪贴板自动粘贴到任意输入框。

---

## 核心功能

### 1. 快捷键触发

| 模式 | 操作方式 | 适用场景 |
|------|----------|----------|
| **按住模式** | 按住快捷键说话，松开结束 | 短句快速输入 |
| **切换模式** | 按一下开始，再按一下结束 | 长文连续输入 |

- 全局快捷键（如 `Ctrl+Shift+V` 或自定义）
- 支持自定义快捷键组合

### 2. 实时语音转写

- **显示方式**：类似微信电脑版语音输入
- **输出单位**：按"语义团"输出，非逐字蹦出
- **实时预览**：在浮动框或光标位置实时显示识别结果
- **自动标点**：语音转文字时自动添加标点符号

### 3. 多风格润色（核心卖点）

松开快捷键后，对整段文字进行润色，力度可调：

| 风格 | 说明 | 示例 |
|------|------|------|
| **口水化** | 保留口语特征，更自然 | "那个...我觉得吧...这个事情..." |
| **轻度去口水** | 去除冗余词，保持口语感 | "我觉得这个事情..." |
| **深度精炼** | 去除所有口语痕迹 | "我认为此事..." |
| **浓缩版** | 极度精简，信息密度最高 | "此事..." |

**风格可选**：正式、 casual、商务、技术、幽默等

## 技术实现要点

### 输入捕获
- Windows 全局热键（RegisterHotKey）
- 麦克风权限管理
- 音频流实时传输

### 语音识别方案

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **本地 Whisper** | 隐私好、离线可用 | 占用资源、准确率略低 | 敏感环境、离线场景 |
| **Kimi K2.5 API** | 准确率高、支持润色 | 需联网、有费用 | 日常办公 |
| **Azure Speech** | 稳定、支持中文 | 微软生态绑定 | 企业环境 |
| **讯飞 API** | 中文强、实时性好 | 国产绑定 | 中文用户 |

### 输出注入
- **方案 A（推荐）**：剪贴板中转 + 模拟粘贴
  - 优点：通用性强，适用于任何输入框
  - 缺点：会覆盖用户剪贴板内容（需临时保存和恢复）
  
- **方案 B（高级）**：UI Automation / SendMessage
  - 优点：不污染剪贴板
  - 缺点：兼容性差，部分应用不支持

### 润色模型调用

```python
# 伪代码示例
def polish_text(text, style="精炼", intensity=3):
    prompt = f"""
    将以下口语化文字转换为{style}风格：

    原文：{text}
    
    要求：
    - 去除冗余口头禅
    - 添加合适标点
    - 保持原意
    - 输出仅包含润色后文字
    """
    return call_llm_api(prompt)
```

## 竞品参考

| 产品 | 特点 | 不足 |
|------|------|------|
| 微信 PC 语音输入 | 实时显示、操作简单 | 无法跨应用、无润色 |
| 讯飞输入法 | 准确率高、功能全 | 输入法形式，需切换 |
| Windows 自带语音 | 系统集成 | 准确率低、无润色 |
| Otter.ai | 专业转写 | 需手动复制粘贴 |

**差异化卖点**：
- ✅ 全局快捷键，不依赖特定应用
- ✅ 一键润色，多种风格可选
- ✅ 本地/云端模型智能切换
- ✅ 剪贴板自动粘贴，零操作成本
